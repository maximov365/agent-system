# Summary

Briefly describe the feature in 2–4 sentences.

Explain:

- what the feature does
- who benefits from it
- what capability it adds to the system

Avoid implementation details.

Example:

> Allow users to export processed text in Markdown format.
> The export should preserve Frank-method formatting and be readable in common Markdown editors.

# Problem

Describe the problem this feature solves.

Explain:

- why the feature is needed
- what limitation currently exists
- what improves after the feature is implemented

Example:

- Currently processed text cannot be easily exported.
- Users need a simple format that works in editors and note-taking tools.
- Markdown provides a portable and widely supported format.
